A Breve História da Web

Tudo começou com Tim Berners-Lee, um cientista britânico que inventou a World Wide Web em 1989. A ideia era criar um sistema para compartilhar informações entre cientistas em universidades e institutos ao redor do mundo. O primeiro website foi ao ar em 1991. Ele usava uma linguagem de marcação simples chamada HTML.

"O potencial da Web é que qualquer coisa pode ser ligada a qualquer outra coisa." - Tim Berners-Lee

Os Três Pilares da Web

Para construir qualquer site moderno, três tecnologias fundamentais que trabalham juntas são fundamentais:

  1. HTML: A linguagem de marcação que define a estrutura e o conteúdo. É o esqueleto da página.
  2. CSS: A linguagem de estilos que controla a aparência visual. É a roupa e a maquiagem.
  3. JavaScript: A linguagem de programação que adiciona interatividade e comportamento. São os músculos e o cérebro.

Estrutura Base: O Esqueleto HTML

Toda página HTML segue uma estrutura fundamental, que funciona como o esqueleto que sustenta todo o conteúdo. É o ponto de partida obrigatório para qualquer projeto web.

"O código inicial que toda página deve ter (boilerplate):

<!DOCTYPE html>
<html lang="pt-br">

  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Título da Minha Página</title>
  </head>

  <body>
    <h1>Bem-vindo ao meu site!</h1>
    <p>Parágrafos, imagens, links e todo o resto do conteúdo visível.</p>
  </body>

</html>

O que está em <head> são as "instruções para a máquina", e o que está em <body> é o "conteúdo para as pessoas".

Glossário de Tags Essenciais

Abaixo está uma tabela com as tags ensinadas no curso, explicando sua função.
Entender a semântica de cada uma é crucial!

Tag Função Exemplo de Uso
<h1> a <h6> Títulos e subtítulos, definindo a hierarquia do conteúdo. <h1>Título Principal</h1>
<p> Define um parágrafo de texto. <p>Isso é um parágrafo.</p>
<strong> Deixa o texto em negrito (indica forte importância ou urgência). Texto importante.
<i> Marca um texto em itálico, geralmente para termos técnicos ou em outro idioma. A palavra de facto.
<u> Sublinha o texto. Deve ser usado com cautela, pois pode ser confundido com um link. Texto sublinhado.
<sup> / <sub> Sobrescrito (ex: 103) e Subscrito (ex: H2O). Fórmulas e notas de rodapé.
<abbr> Define uma abreviação, com a expansão visível ao passar o mouse. CSS
<blockquote> Usada para citar um bloco de texto de outra fonte. Citações longas e destacadas.

HTML Semântico: Dando Significado à Estrutura

HTML semântico significa usar as tags corretas para o trabalho que elas foram criadas para fazer. Em vez de usar <div> para tudo, usamos tags que descrevem o conteúdo, como <header>, <nav>, <main>, <section> e <footer>.

Isso ajuda os navegadores, os mecanismos de busca (SEO) e, principalmente, as tecnologias assistivas (como leitores de tela) a entenderem a página.

"Qualquer tolo consegue escrever código que um computador entende. Bons programadores escrevem código que humanos conseguem entender."

— Martin Fowler

Acessibilidade: Construindo uma Web para Todos

Pense em uma rampa ao lado de uma escada em um prédio. A rampa não serve apenas para cadeirantes, mas também para pessoas com carrinhos de bebê, malas pesadas ou um gesso no pé. A rampa torna o acesso mais fácil para todos.

Acessibilidade na web segue o mesmo princípio: é a prática de remover barreiras e garantir que os sites possam ser navegados e compreendidos por todas as pessoas, incluindo aquelas com deficiências visuais, auditivas, motoras ou cognitivas.

Na Prática: Como o HTML Ajuda?

Um HTML bem estruturado e semântico é a fundação de um site acessível. Aqui estão alguns exemplos chave:

  • Uso Correto das Tags (Semântica): Usar <nav> para navegação, <button> para botões e <main> para o conteúdo principal informa aos leitores de tela exatamente qual o propósito de cada parte da página. Isso fornece um roteiro claro para as tecnologias assistivas, que podem anunciar 'Início da navegação' em vez de apenas 'Início de um grupo de elementos'.
  • Imagens Descritivas (Atributo alt): O atributo alt é a "voz" da sua imagem para quem não pode vê-la. Em vez de um nome de arquivo, ele deve descrever o que a imagem mostra. 
    <!-- Ruim: --> <img src="gato.jpg" alt="foto">
    <!-- Bom: --> < img src="gato.jpg" alt="Um gato preto de olhos verdes sentado na grama em um dia ensolarado.">
  • Links que Fazem Sentido: O texto de um link deve ser descritivo. Um leitor de tela pode listar todos os links de uma página. Uma lista de "clique aqui" não informa nada, mas uma lista de "Leia nosso relatório" é perfeitamente clara.

"O poder da Web está em sua universalidade. O acesso por todos, independentemente de deficiência, é um aspecto essencial."

— Tim Berners-Lee

Formulários: A Interação com o Usuário

Formulários são essenciais para coletar dados do usuário, como em cadastros e logins. A boa prática é sempre associar uma <label> a cada campo de entrada (<input>) para melhorar a acessibilidade e a usabilidade.

Comentários

Comentários em HTML são trechos de código que o navegador ignora completamente. Eles são extremamente úteis para deixar notas para você mesmo ou para outros desenvolvedores, explicar uma parte complexa do código ou desativar temporariamente uma linha sem apagá-la.

<!-- Este é um comentário. Ele não aparecerá na página. -->

Atenção: Nunca coloque informações sensíveis (como senhas ou dados de usuários) dentro de comentários, pois eles ainda são visíveis para qualquer um que inspecione o código-fonte da página.

Cuidado com o Obsoleto!

Antigamente, usava-se tags para definir a aparência do site. A tag <font> é o exemplo mais famoso. Hoje, isso é uma má prática. A estilização deve ser feita com CSS.

Forma antiga (não faça isso!): <font color="blue">Texto azul</font>
Forma moderna (o correto!): Criar uma classe no CSS e aplicá-la no HTML.
